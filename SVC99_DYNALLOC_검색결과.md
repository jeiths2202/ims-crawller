## 🔥 주요 이슈 #350137 (진행중 - HIGH Priority)

### 📌 이슈 개요

**제목**: [Project] [일본 동경해]SVC99(DYNALLOC) 동작 이슈에 대한 패치 제공 요청
**상태**: Assigned (처리 중)
**우선순위**: High
**담당자**: 이민성 연구원
**제품**: OpenFrame ASM v4 1676 32bit
**환경**: RedHat 9.4 OpenFrame 32bit

### 🐛 이슈 내용

**문제 현상**:
- SVC99 매크로(DYNALLOC) 실행 시 **상세 리턴코드가 NULL로 설정**되는 문제
- 호스트 환경에서는 정상 동작하나 OpenFrame 환경에서는 비정상 동작
- IDCAMS 실행 결과에 따른 상세 리턴코드가 정상적으로 세트되지 않음
- 원래 정상 종료되어야 할 시나리오가 **에러 처리로 분기**되는 현상 발생

**발생 시나리오**:
1. 데이터셋 삭제용 카탈로그 프로그램(DADEL)에서 ASM 프로그램 PZS555 실행
2. PZS555 내부에서 IDCAMS 호출
3. 삭제 대상 데이터셋이 존재하지 않아 IDCAMS가 RC≠0으로 종료 (정상)
4. PZS555는 IDCAMS의 RC≠0일 경우 SVC99(DYNALLOC)를 호출하여 상세 리턴코드 확인
5. **호스트**: 상세 리턴코드 정상 세트 → 정상 종료 처리
   **OpenFrame**: 상세 리턴코드 NULL → **에러 처리로 종료**

**기술적 원인** (김도환 PS 답변):
- OFASM에서는 DYNALLOC 기능을 지원하고 있으나,
- 현재 시나리오에서 사용중인 DYNALLOC 기능은 allocation 시 **DALRTORG를 요구**
- DALRTORG는 **현재 비호환 사항**으로 지원하고 있지 않아 프로그램 종료됨
- IBM 문서: https://www.ibm.com/docs/en/zos/3.1.0?topic=functions-dsorg-return-specification-key-0057

> 예를 들어, DSCB 자체를 지원하지 않는 구조적 제약이 있는 것은 아닌지?
